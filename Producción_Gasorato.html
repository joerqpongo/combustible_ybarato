Guía de Despliegue en Producción

Pasos para la puesta en marcha del buscador en servidores AWS EC2 o VPS Linux.

Configuración de Red Inicial

⚠️ Control de Tráfico en AWS

Por defecto, AWS bloquea todo el tráfico entrante salvo el puerto 22 (SSH). Es obligatorio configurar el Grupo de Seguridad para permitir el acceso web.

Sigue estos pasos en la consola web de AWS antes de configurar el servidor:

  1. Ve a la consola de AWS EC2 y selecciona tu instancia en ejecución.
  2. En las pestañas de detalles inferiores, accede a Security (Seguridad) y haz clic en tu Security Group (Grupo de Seguridad) asignado.
  3. Haz clic en el botón Edit inbound rules (Editar reglas de entrada).
  4. Añade una nueva regla según la estrategia de despliegue que prefieras:
    • Para Docker (Puerto 80): Tipo HTTP | Puerto 80 | Origen Cualquier lugar IPv4 (0.0.0.0/0)
    • Para Servicio Nativo (Puerto 8501): Tipo TCP personalizado | Puerto 8501 | Origen Cualquier lugar IPv4 (0.0.0.0/0)
  5. Haz clic en Save rules (Guardar reglas) para aplicar los cambios en el cortafuegos.

Opción 1: Despliegue con Docker (Recomendado)

Esta metodología utiliza un contenedor aislado optimizado, evitando conflictos con versiones locales de Python o librerías del sistema.

  1. Conexión al servidor vía SSH: 
    ssh -i "tu_clave.pem" ubuntu@tu_ip_publica_aws
  2. Instalación y activación de Docker:

    Si es un servidor nuevo, ejecuta la instalación base:

    sudo apt update
sudo apt install -y docker.io
sudo systemctl enable --now docker

# Opcional: Ejecutar docker sin sudo (requiere reiniciar sesión SSH)
sudo usermod -aG docker $USER
  3. Clonación del repositorio del proyecto: 
    git clone https://github.com/joerqpongo/combustible_ybarato.git
cd combustible_ybarato
  4. Construcción de la imagen: 
    docker build -t combustible-app .
  5. Ejecución del contenedor en segundo plano:

    Mapeamos el puerto interno 8501 (Streamlit) al puerto estándar HTTP 80 del VPS.

    docker run -d \
  --name combustible-buscador \
  -p 80:8501 \
  --restart always \
  combustible-app

Opción 2: Servicio Nativo del Sistema (Systemd)

Ideal si prefieres prescindir de capas de virtualización y deseas que Linux controle el proceso de la aplicación directamente como un servicio nativo.

  1. Entorno de ejecución y dependencias: 
    git clone https://github.com/joerqpongo/combustible_ybarato.git
cd combustible_ybarato

# Preparación del entorno virtual de Python
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
  2. Creación del archivo de configuración de Systemd: 
    sudo nano /etc/systemd/system/combustible.service

    Copia y pega el siguiente contenido dentro del editor. Asegúrate de verificar las rutas absolutas si tu usuario no es ubuntu:

    [Unit]
Description=Buscador de Combustible Streamlit App
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/combustible_ybarato
ExecStart=/home/ubuntu/combustible_ybarato/venv/bin/streamlit run app.py --server.port 8501 --server.address 0.0.0.0 --server.headless true
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
  3. Activación e inicio del servicio: 
    # Notificar al sistema el nuevo archivo
sudo systemctl daemon-reload

# Habilitar el inicio automático con el sistema
sudo systemctl enable combustible

# Arrancar la aplicación inmediatamente
sudo systemctl start combustible

🛠️ Operaciones de Mantenimiento rápido

Utiliza estos flujos de comandos una vez el sistema se encuentre operando en producción para realizar diagnósticos o actualizaciones.

Gestión con Docker

Estado: docker ps

Logs en vivo: docker logs -f combustible-buscador

Actualizar Versión:

git pull
docker build -t combustible-app .
docker rm -f combustible-buscador
docker run -d --name combustible-buscador -p 80:8501 --restart always combustible-app

Gestión con Systemd

Estado: sudo systemctl status combustible

Logs en vivo: journalctl -u combustible.service -f

Actualizar Versión:

git pull
source venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart combustible
